Angular Pipes

Pipes are used to format the data before displaying in the view . Pipe is used by using | . This symbol is called pipe operator
Lets see some of built-in pipes and how they work

Date pipe : "expression | pipename"

Unformatted date : {{toDate }}

Formatted date : {{toDate | date}}

Passing Arguments to the pipe
We can also pass optional arguments to the pipe. The arguments are added to the pipe using a colon (:) sign followed by the value of the argument. If there are multiple arguments separate each of them with the colon (:). For example, we can pass the format as the argument to the date pipe, which is Optional. The medium is one of the valid value of the format argument, which displays the date in yMMMdjms format.

Formatted date : {{toDate | date:'medium'}}

short : {{toDate | date:'short'}}

fullDate : {{toDate | date:'fullDate'}}

longDate : {{toDate | date:'longDate'}}

mediumDate : {{toDate | date:'mediumDate'}}

shortDate : {{toDate | date:'shortDate'}}

mediumTime : {{toDate | date:'mediumTime'}}

dd-MM-y : {{toDate | date:'dd-MM-y'}}

dd-MM-yy HH:mm : {{toDate | date:'dd-MM-yy HH:mm'}}

Pipes Chaining
Pipes can be chained together to make use of multiple pipes in one expression. For example in the following code, the toDate is passed to the Date Pipe. The output of the Date pipe is then passed to the uppercase pipe.

{{toDate|date|uppercase}}

Angular Built-in Pipes
The Angular has several built-in pipes, which you can use in your applicationbr

Some of the important pipes are Date Pipe, Uppercase Pipe, Lowercase Pipe, Number Pipe/ Decimal Pipe, Currency Pipe ,Percent Pipe

UperCase & LowerCase pipe & TitleCase pipe
As the name suggest these pipes transform the string to Uppercase or lowercase

Unformatted :{{msg}}

Uppercase :{{msg | uppercase}}

Lowercase :{{msg | lowercase}}

Titlecase :{{msg | titlecase}}


Slice pipe
Creates a new List or String containing a subset (slice) of the string or array.
array_or_string_expression | slice:start[:end]
array_or_string_expression: is the string to slice
start: is the start position/index from where the slicing will start
end: is the ending index/position in the array/string

Complete String :{{msg}}

Example 1 :{{msg | slice:11:20}}

Example 2 :{{msg | slice:-9}}
The slice pipes take two arguments. The first argument start is the starting index of the string/array. The second argument end is the ending index of the string/array. If the start or end index is negative then the index is counted from end of the string/array

Currency Pipe
Formats a number as currency using locale rules.

number_expression | currency[:currencyCode[:symbolDisplay]]

currencyCode: is the ISO 4217 currency code, such as USD for the US dollar and EUR for the euro.
symbolDisplay is a boolean indicating whether to use the currency symbol or code. Use true to display symbol and false to use code

Unformatted :{{cur}}

Example 1 :{{cur | currency }}

Example 2 :{{cur | currency:'INR':true}}

transforming the first text in uppercase using custom-pipe

Unformated text : {{text}}
