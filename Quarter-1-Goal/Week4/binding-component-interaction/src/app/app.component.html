Data Binding

The way through which a component class and a HTML DOM (template) can interact with each other is called data binding.
In Angular there are two types of data binding
  • One way binding
  • Two way data binding
One way Bindingis a one-way interaction between component and its template. If you perform any changes in your component, then it will reflect the HTML elements.You can achieve one way binding by these ways
String Interpolation(FROM COMPONENT TO VIEW) :In Angular, Interpolation is used to display data from component to view (DOM). It is denoted by the expression of double closed curly braces and also known as mustache syntax

Property Binding Property binding is an example of one-way databinding where the data is transferred from the component to the view. The main advantage of property binding is that it facilitates you to control elements property. Every HTML has some its own property for example button element has disabled property, img has src , anchor(a) has href, input has value etc.
The Property binding allows us to bind HTML element property to a property in the component. Whenever the value of the
component changes, the Angular updates the element property in the View. You can set the properties such as class, href, src, textContent, etc using property binding

Event BindingTo bind to an event you use the Angular event binding syntax. This syntax consists of a target event name within parentheses to the left of an equal sign, and a quoted template statement to the right
Event binding example

1. Show a alert msg on click event (click)="show()"

2. Disable a button after click [disable]="flag"

3. Update the username (input)="updateUsername($event)

ElementRef & NativeElement

** ElementRef is a wrapper that wraps a specified DOM element and allows us to work with the DOM, it provides the nativeElement object that exposes all the methods and properties of the native element(all the html element i.e div,span,p,h3 etc..)

** Elementref is used to access the DOM element and manipulates it. It has the nativeElement property that contains all the properties of the DOM element

**ElementRef is simply like document.getElementById("any_id")

@ViewChild & @ViewChildren

**To get access to DOM element , child component or directive from parent component we use @ViewChild Decorator NOTE: @ViewChild returns only the first element of the matching selector/template reference variable

1. **First use of ViewChild :Gettting DOM element using @ViewChild decorator & Template reference variable type something in the below input box and hit the show-name button

2. **Second use of @ViewChild : Calling the child-component method in parent-component using @ViewChild decorator & Template reference variable

** Unlike @ViewChild , the @ViewChildren decorator is used for accessing multiple elements . Its return type is QueryList

NOTE : @ViewChildren returns all the element of the matching selector/template reference variable

**QueryList is just a name for an object that stores a list of items. QueryList will always be updated whenever any child element added,updated, or removed from the HTML DOM tree.
